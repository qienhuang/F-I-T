# Langton's Ant实验特别说明

## 重要提示

Langton's Ant的行为与Conway's Game of Life有显著不同：

### 典型行为模式

1. **早期阶段 (0-10,000步)**：看似随机的混沌行为
2. **transition (约10,000步)**：开始建造"高速公路"
3. **晚期阶段 (10,000+步)**：沿对角线方向稳定移动

### 为什么实验可能"失败"

**这不是bug！** Langton's Ant的FIT命题测试比Conway更具挑战性，原因包括：

1. **需要极长演化时间**
   - 标准配置需要20,000-25,000步
   - 每次运行约需2-3分钟

2. **高速公路特征微妙**
   - 不是"完美对齐"而是对角螺旋移动  
   - 对齐度阈值需要放宽（0.7而非0.9）
   - 应同时检测净位移而非只看对齐度

3. **单次运行的随机性**
   - 虽然规则确定，但某些初始配置可能需要更长时间
   - 建议至少运行3-5次取平均

### 推荐的测试策略

#### 快速验证（5-10分钟）
```python
num_runs: int = 3
max_steps: int = 20000
```

#### 标准验证（15-25分钟）
```python
num_runs: int = 5  
max_steps: int = 25000
```

#### 完整验证（40-60分钟）
```python
num_runs: int = 10
max_steps: int = 30000
```

### 如何判断结果

**不要期待100%通过率！** Langton's Ant的命题测试更多是探索性的：

- **P1 (持久性)**: 高速公路一旦出现应该保持稳定
  - ≥60%通过 = 支持
  
- **P3 (力方差衰减)**: 进入高速公路后力方差应下降
  - ≥50%显示指数衰减 = 支持
  
- **P11 (相变信号)**: 应检测到混沌→高速公路的转变
  - ≥50%检测到 = 支持
  
- **P18 (时间尺度分离)**: 较难在Langton中验证
  - 任何清晰的分离都是支持性证据

### 调试技巧

如果所有测试都失败：

1. **运行诊断脚本**：
   ```bash
   python langton_diagnostic.py
   ```
   
2. **检查是否检测到高速公路**：
   - 诊断输出应该显示"Net Displacement"在15000步后超过300
   
3. **调整阈值**：
   - 如果从不检测到高速公路，降低`alignment_threshold`到0.5
   - 增加`max_steps`到30000

4. **可视化（如果可能）**：
   - 高速公路应该是一条对角线模式
   - 蚂蚁应该在对角线上持续移动

### 与文献对比

标准Langton's Ant文献报告：

- 高速公路通常在~10,000步出现（±2000步）
- 高速公路以2格/2步的速度对角移动
- 这是一个强二级相变（first-order transition）

FIT框架的预测：

- **P11应该通过**：这是最清晰的相变案例之一
- **P3可能部分通过**：衰减可能不是完美指数
- **P1取决于检测精度**
- **P18最具挑战性**

### 成功案例

如果您的实验显示：

1. 诊断脚本检测到高速公路（~10k-15k步）
2. P11检测到相变信号
3. P3显示某种形式的力方差下降

那么**FIT框架在Langton's Ant上得到了合理支持**，即使不是所有命题都严格通过。
