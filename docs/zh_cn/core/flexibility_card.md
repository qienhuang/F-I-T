# Flexibility Card (v2.4.1+)

**Artifact type**: Core Artifact (handoff-ready)

**Purpose**: 用可操作的方式定义并监控“灵活性”（Flexibility），使系统在进入稳定/协调相（例如 Φ₃）后仍具备可控的适应能力，并能区分：
- 合法的相变/重组（phase transition / reconfiguration）
- 有害的锁死（lock-in）、漂移（drift）
- 纯测量问题（estimator instability）

**FIT/EST 前提**：所有判定都绑定到显式估计器元组 $ \mathcal{E} $ ，并且在 EST 下只接受“可审计的可采纳估计器族”。

**范围说明**：本卡片不引入任何新原语或新命题；它只是 FIT v2.4/v2.4.1 之下的操作化分类与监测方案。


$$
\mathcal{E} = (S_t, \mathcal{B}, \{\hat{F},\hat{C},\hat{I}\}, W)
$$

---

## 三类灵活性

> 读法：每一类都给出“对应命题（来自 FIT v2.4 / v2.4.1）→ 对应指标 → 三类失败模式（锁死/漂移/估计器不稳）”。

---

## 1) 结构灵活性（Constraint / Phase Flexibility, C-Flex）

**定义**：在不破坏“相内一致性”的前提下，允许约束代理 $ \hat{C}(t) $ 在相变附近发生短期重组/回撤，并能通过可控方式完成“约束重配置”（constraint reconfiguration），以支持新信息/新结构的形成。

**对应命题**
- **P2a**（相内单调性）：在显式相上下文中，$ \hat{C}(t) $ 在同一相内统计上非递减。
- **P2b**（晚相不可逆）：进入协调结构相后，大尺度结构回撤变得快速不太可能（概率不可逆，而非“完全不变”）。
- **P12**（信息增长需要约束重配置）：若 $ I_{useful} $ 在固定边界下持续增长，则必须伴随 $ C $ 的增长或重配置。

**对应指标（最小集）**
- **相内单调性违例率**：
  - $ v_C := |\{t \in \Phi_k: \hat{C}(t+1) < \hat{C}(t)\}| / |\Phi_k| $ （按相分段统计）
- **相变信号强度**：
  - $ J(t) := |\mathrm{d}(\hat{I}/\hat{C})/\mathrm{d}t| $ 或变化点检测得分（用于识别“重组窗口”）；实现必须预注册，并声明平滑/差分窗口 $W_s$
- **重配置可达性**：
  - 在预注册“释放/重配置协议” $ \Pi_{reconf} $ 下，能否在有限步数内形成新的稳定相（例如从 Φ₃ → Φ₂/Φ₃'）

**失败模式**
- **锁死（Lock-in）**：
  - 症状：长期 $ \hat{C}(t) $ 饱和、$ J(t) \approx 0 $，且外部任务/环境指标显示需要结构变更；执行 $ \Pi_{reconf} $ 后仍无法进入新相。
  - 典型后果：系统“稳定但不再可用”，只能通过外部强制放松边界/约束才能变化。
- **漂移（Drift）**：
  - 症状：在同一相内出现持续性 $ \hat{C}(t) $ 下行（$ v_C $ 远超阈值），或 $ I/C $ 长期走弱而非“相变型重组”。
  - 典型后果：约束被侵蚀，结构协调性下降。
- **估计器不稳（Estimator instability）**：
  - 症状：相分段/相变点对不同可采纳 $ \hat{C} $ / $ \hat{I} $ 版本高度不一致；或对窗口 $ W $ 的微小改变导致结论翻转。
  - 处理：标注 `ESTIMATOR_UNSTABLE`，禁止把现象当作 P2/P12 的支持/挑战。

## 2) 动力灵活性（Force / Policy Flexibility, F-Flex）

**定义**：系统在扰动或目标变化时，能够产生“足够的驱动力”完成必要迁移，同时在稳定阶段维持低 $ \sigma^2(F) $（避免长期高力噪声导致漂移）。它回答：“既能稳住，也能动起来”。

**对应命题**
- **P5**（恢复时间与约束）：在同一系统内，更高约束的吸引子应更快从小扰动恢复。
- **P18**（近吸引子时间尺度分离）：近吸引子时 $ \tau_F \ll \tau_C $，快变量先稳住，慢变量缓慢演化。
- （作为边界条件）**P1**（nirvana 不可逆）：若没有约束放松，进入低 $ \sigma^2(F) $ 的强吸引子后“自发退出”概率趋零。

**对应指标（最小集）**
- **力方差与其回落**：
  - $ \sigma^2(F_t) $ 的基线、扰动后峰值、回落速率（指数/幂律拟合均可，但需预注册）
- **恢复时间**：
  - 在预注册扰动协议 $ \Pi_{perturb} $ 下，$ \tau_{recover} := \min\{\Delta t: \text{回到阈值带}\} $
- **时间尺度比**：
  - $ r_{ts} := \tau_F / \tau_C $（用一致的窗口法/变化点法估计）

**失败模式**
- **锁死（Lock-in）**：
  - 症状：$ \sigma^2(F) \approx 0 $ 且 $ \tau_{recover} $ 对“迁移型扰动”近似无穷（系统只会回到原点，不会迁移到新吸引子）。
  - 典型后果：在目标变化时无法重定向，只能通过显式约束放松/边界改变解锁。
- **漂移（Drift）**：
  - 症状：$ \sigma^2(F) $ 长期偏高且无稳定平台（P4 意义上的 plateau 不出现），或 $ r_{ts} $ 失去分离（快变量也长期不稳）。
  - 典型后果：系统持续游走、表现不稳定。
- **估计器不稳（Estimator instability）**：
  - 症状：$ \hat{F} $ 在等价表示/窗口变化下波动剧烈，导致 $ \tau_{recover} $、$ r_{ts} $ 不可复现。
  - 处理：先走 EST 的可采纳性与一致性门槛；若门槛失败，直接标注 `ESTIMATOR_UNSTABLE`。

## 3) 认识灵活性（Estimator / Interpretation Flexibility, E-Flex）

**定义**：在不落入“后验换尺救理论”的前提下，允许针对不同相/不同任务类型切换或扩展估计器族，并保持可审计、可复现、可交接。

**对应命题 / 规则**
- **EST 可采纳性（A1–A8）**：估计器必须声明作用域、稳健性、任务类型、复杂度成本与预注册等。
- **P10（任务类型一致性门）**：对 ordinal/metric/topological 任务分别使用相应一致性门槛（秩相关、阈值对齐、事件结构一致性）。
- **v2.4.1 约束**：任何 P2/P17 结论必须声明相上下文与估计器作用域；相变附近允许代理重组，不默认当作反例。

**对应指标（最小集）**
- **一致性门槛通过率**：
  - Ordinal：中位 Spearman/Kendall 相关 ≥ 预注册阈值
  - Metric：阈值事件对齐（$ |t_1^* - t_2^*| \leq \Delta t_{max} $）+ 斜率/残差稳定
  - Topological：相分段数量/顺序一致 + 变化点容差
- **估计器族稳健性（robustness pass rate）**：
  - $ \text{pass\_rate} := |\{\mathcal{E} \in \mathfrak{E}_{adm}: P_i[\mathcal{E}] \text{ supported}\}| / |\mathfrak{E}_{adm}| $
- **窗口敏感度**：
  - $ \|\hat{X}(t; W+\delta W) - \hat{X}(t; W)\| $ 的上界或经验斜率（$ X \in \{C,I,F\} $）

**失败模式**
- **锁死（Lock-in）**：
  - 症状：测量管线被单一表示/单一估计器锁定，导致相变被误读为“理论失败”或“系统崩坏”，无法进行相内/相间的正确归类。
- **漂移（Drift）**：
  - 症状：未预注册即更换估计器/阈值；或为了得到期望结论在估计器族里挑选“最好看的那一个”。
  - 典型后果：不可复现、不可交接，等价于测量层面的制度性漂移。
- **估计器不稳（Estimator instability）**：
  - 症状：一致性门槛失败；同一数据在可采纳估计器族下给出相互冲突的相划分/结论。
  - 处理：按 EST 标记 `ESTIMATOR_UNSTABLE`，优先修测量而不是修理论。

---

## 使用约定（交接要点）

1. **先分相，再谈单调**：任何 “$ C(t) $ 是否单调” 判断必须是 *相内* 判断（P2a），相变窗口内允许重组。
2. **锁死与不可逆不同**：P2b/P1 描述的是概率不可逆；“灵活性”来自被设计/被允许的约束放松与重配置协议，而不是指望系统自发逃逸。
3. **任何争议先过 P10/EST**：只要一致性门槛不过，就停止讨论“支持/挑战”，先把问题归类为测量不稳。
