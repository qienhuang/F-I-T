# PATTERN_02 — 反馈主导（Feedback Dominance）

## 1) 一句话总结

一个快速反馈回路（通常是算法回路）成为系统演化的主驱动，压制更慢的纠错通道，使系统有效设计空间收缩到“回路能优化的东西”。

## 2) 适用场景

- 你看到对某个指标的加速收敛（“回路测什么，就朝什么长”）。
- 系统对回路规则极其敏感（政策/排序/奖励函数轻微变化就翻车）。

## 3) 结构图（最小）

```mermaid
flowchart LR
  M[可测代理指标 / 奖励] --> L[快速反馈回路]
  L --> P[策略 / 行为]
  P --> O[结果]
  O -->|弱/慢| C[纠错通道]
  C -->|太晚| M
```

## 4) 定性签名（PT-MSS 风格）

- 接近回路时间尺度的快速振荡或失控。
- “Goodhart”症状：指标变好但下游结果变差。
- 放慢回路/引入慢通道后，可控性恢复。

## 5) 最小干预（非处方）

- 增加能覆盖快速回路的慢评估通道（可否决）。
- 多目标/鲁棒性约束，避免代理指标坍缩。
- 引入回路不可见的 holdout/checkpoint。

## 6) 例子

**正例（示意）：**
- 智能手机与注意力系统：`docs/zh_cn/cases/CASE_01_Phone_Attention_System.md`

**反例（边界）：**
- 若纠错通道强且及时（例如硬安全约束），即便使用指标，也可能不会进入反馈主导。
