# PATTERN_03 — 约束饱和（Constraint Saturation / C-wall）

## 1) 一句话总结

系统可达状态空间持续收缩，直到撞上“约束墙”：之后再加大压力只能带来边际收益递减、脆弱性上升或病态副作用。

## 2) 适用场景

- 你观察到停滞：投入/压力增加，但结果不再改善。
- 副作用上升：系统开始“钻空子”或伤害长期可持续性。

## 3) 结构图（最小）

```mermaid
flowchart LR
  F[驱动力 / 优化压力] --> X[状态探索]
  X -->|收缩| S[可达状态空间]
  S --> W[约束墙]
  W --> D[边际递减 / 脆弱性]
```

## 4) 定性签名（PT-MSS 风格）

- 主结果平台期，但成本/风险上升。
- 对微小扰动更敏感（脆弱平衡）。
- 放松某些约束或引入新自由度后，出现新阶段。

## 5) 最小干预（非处方）

- 扩展可达状态空间（新自由度、松弛、模块化）。
- 降低硬耦合；把约束显式化并在可能时“可谈判化”。
- 从“推更大力”转向“改变约束几何”。

## 6) 例子

**正例（示意）：**
- 内容平台与内卷：`docs/zh_cn/cases/CASE_02_Content_Platform_Involution.md`

**反例（边界）：**
- 松弛度高、耦合低的系统，即便压力大，也可能不显著饱和。
