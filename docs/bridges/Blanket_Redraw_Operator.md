# The blanket-redraw operator

## Status

Operator proposal under validation.

This text defines a FIT-compatible way to talk about boundary reconfiguration without pretending that the current bridge experiments have already isolated such events uniquely.

## Reader boundary

This note should be read as:

- a formal target
- a vocabulary proposal
- an admissibility-oriented bridge note

It should not be read as a settled event detector or an established cross-domain mechanism.

## Core idea

Many intelligence-relevant transitions are not well described by fixed-boundary regulation alone.

Examples include:

- external memory coupling
- tool extension
- delegation
- institutional scaffolding

The proposed operator language is:

> boundary redraw -> constraint-topology rewrite -> reachable state-space reconfiguration

This is a useful formal target for FIT-compatible bridge work.

## What current evidence supports

Current evidence supports only a narrow statement:

> known-positive grokking-source runs are distinguishable from the tested controls by a denser sustained trigger regime.

This is compatible with structural reorganization language.

## What current evidence does not support

Current evidence does **not** establish:

- that the measured signal uniquely corresponds to blanket redraw
- that blanket redraw has been localized as an event moment
- that the operator is already validated across domains

So the correct current use of this note is:

- operator proposal
- admissibility guide
- experimental target specification

## Why keep this note

The value of this note is not that it proves blanket redraw.

Its value is that it gives FIT-compatible language for:

- discussing boundary change
- separating fixed-boundary regulation from boundary rewrite
- specifying what a stronger future validation would need to show

## Placement

This note belongs under a bridge-facing area such as `docs/bridges/`.

It should be read as a working operator note, not as an empirically settled mechanism.
