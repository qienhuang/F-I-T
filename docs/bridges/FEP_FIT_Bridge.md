# FEP as a Φ₃ engine inside FIT

## Status

Bridge note.

This text is intended as an interface specification between FEP-style regulation and FIT-style phase analysis.

It should be read as a **conditional bridge rule**, not as a settled empirical theorem.

## Reader boundary

This note should be read as:

- bridge-facing
- bounded by current evidence
- non-core

## Core claim

Within the current FIT reading:

- FEP is most naturally interpreted as a **within-regime engine**
- FIT is needed for **phase structure**, **boundary redraw**, and **transition diagnostics**

The proposed relationship is therefore:

> FEP-like dynamics may be admissible inside FIT analysis under sufficiently stable `Φ₃`-like boundary conditions.

This is a governance claim about when a cross-framework mapping is legitimate.

It is not a claim that current experiments have already established universal containment.

## What current evidence supports

Current bridge experiments support a narrow statement:

> Under the tested grokking-source setup, known-positive runs show a denser sustained trigger regime than the tested controls.

This supports a **regime-classifier** interpretation.

## What current evidence does not support

Current evidence does **not** support:

- localization of the generalization transition moment
- a completed theorem of the form `FEP ⊂ FIT`
- a full empirical identification of `Φ₃-engine` dynamics

So the safest current interpretation is:

> FEP-like descent appears to function as a narrower within-regime mechanism, while FIT-style descriptors remain necessary for boundary-level reorganization and transition analysis.

## Why keep this note

This note is useful because it gives a disciplined answer to a recurring confusion:

- FEP is good at describing regulation under an already-usable boundary
- FIT is needed when the boundary itself, the admissible state space, or the phase grammar changes

That distinction is already theoretically valuable even before stronger bridge validation arrives.

## Falsification posture

This note should be revised downward if future bridge experiments show either:

1. no stable regime separation relative to tested controls
2. no useful distinction between FEP-like within-regime signals and FIT-style transition descriptors

## Placement

This note belongs in a bridge-facing section such as `docs/bridges/`.

It should not be presented as a FIT core law.
